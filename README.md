# local-code-agent

Локальний кодинг-агент на **gemma4:26b-a4b-it-qat** (Ollama), орієнтований на слабку
локальну модель: вузький контекст, розділення ролей і детерміноване поводження з даними.

## Ідея

- **Planner** (think=on) — читає задачу, ріже її на дрібні кроки, класифікує їх
  (LLM / детермінований / inline).
- **Executor** (think=off, вузький контекст) — виконує один крок над одним файлом.
- **Strip/Restore** — великі опакові літерали (напр. протобафи в коді) вирізаються
  детерміновано в плейсхолдери *до* LLM і повертаються байт-у-байт *після*. Модель
  їх не бачить → не псує, а контекст не роздувається.

## Чому не Aider

Aider завжди кладе весь файл у контекст і регенерує його цілком — на токен-щільних
файлах слабка локальна модель захлинається й псує дані. Тут дані взагалі не йдуть
крізь модель.

## Структура

```
agent/
  config.py   профілі EXECUTOR / PLANNER
  llm.py      клієнт Ollama (tool-calling, think-перемикач)
  literals.py strip / restore           (M1)
  tools.py    детерміновані інструменти  (M1-M2)
  executor.py пайплайн виконання кроку   (M1)
  memory.py   стан задачі на диск        (M3)
  planner.py  декомпозиція задачі        (M4)
```

## Вимоги

- Python 3.12, `requests`
- Ollama з моделлю `gemma4:26b-a4b-it-qat`
